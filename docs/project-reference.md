## Design Direction

- This is a full overhaul mod, not a data-only reskin.
- Pokemon should be added as native `PokemonChampion` entries, with move behavior implemented in Rust where needed.
- Design can be DOTA-like: strong identities, extreme strengths, long cooldown nukes, and intentionally "broken" mechanics are acceptable. Balance will happen through iteration.
- Prefer implementing the requested fantasy faithfully, then tune numbers after in-game tests.
- Balance ethos: Pokemon are allowed to be absurdly strong at one clear job as long as they have real tradeoffs and hard counters. Inteleon can be a terrifying long-range killer if he is fragile when caught; Greninja can be very squishy if he is hard to catch; Blissey can have an enormous health pool if she has low damage/resists and counterplay. Prefer sharpening identity and counter-picks over flattening everyone into safe generalists.
- Type and kit counters matter. Example: Venusaur should be a nightmare matchup for Greninja because Grass is super effective into Water/Dark and Venusaur brings repeated CC/lockdown. Preserve these obvious matchup stories when assigning stats, tools, and draft strategy tags.
- The server-side runtime champion roster filter is disabled. `src\pokemon_roster.rs` is kept as a reference/probe record only. On June 6, 2026, `after_management_tick` crashed after the engine logged `attempt to divide by zero`: first while iterating `Database.pre_patch_data`, then while reading `Database.available_champions`. A later `on_server_start` / `before_management_tick` attempt crashed on Start New Game with Windows fault `pokemon_moba.dll+0xe27e4`, symbolized to `pokemon_roster.rs:88` inside `retain_known_pokemon_only_and_complete` while counting/filtering a roster vector. Do not re-enable `set_server_extension(...)` for direct roster vector reads without a substantially different lifecycle strategy.
- Temporary day-advance crash diagnostics are documented in `docs\crash-probe.md`. The active probe writes panic details and callback panics to `logs\panic-probe.log` and `logs\mod-panic-probe.log`. `logs\roster-probe.log` is not expected while the server roster callback is disabled.
- Local executable strings confirm `GamePlayOption` stores both `champion_pool` and `custom_champions`, with `champion_add_count` as a separate option. Bundled English UI text says Custom "lets you customize the champion pool used in the game," but no public SDK/source evidence has confirmed that future patch additions are guaranteed to stay inside Custom selections. Keep the runtime filter unless in-game patch-cycle testing proves Custom is sufficient.
- Do not mutate `ClientDatabase.champion_positions` from client update hooks. Both broad and guarded attempts caused hard crashes when entering gameplay/start-new-game flows. Position indices remain `Top=0`, `Jungle=1`, `Mid=2`, `Bottom=3`, `Support=4`.
- Pokemon lane intent is stored in `src/pokemon_positions.rs` as a sidecar table. `PokemonDraftScoreHook` nudges AI picks toward missing lane coverage and pick-slot fit without touching the crash-prone client position map. As of the June 2026 game update, staff advice appears to predict likely champion roles, but the SDK still exposes no swap-decision hook; keep pick-slot scoring softer than coverage and matchup scoring so the draft is not locked to Top/Jungle/Mid/Bottom/Support order if the base game can infer swaps. A narrower two-slot cache write from `post_update` also caused CTDs, so do not write `ClientDatabase.champion_positions` from native hooks. Do not put lane recommendations into skill descriptions.
- Pokemon draft strategy is stored in `src/pokemon_strategy.rs` as broad traits, threat tags, and vulnerability tags. Prefer this metadata over one-off pair counters. The draft hook combines type matchup, same-lane strategic matchups, cross-lane team threats, blind-pick counter risk, and team composition penalties such as repeated type/vulnerability stacking.
- Eevee and all Eeveelutions carry `StrategyTrait::Eeveelution` as a trait, threat, and vulnerability. Draft scoring treats this as a modest count-race tiebreaker, not a primary pick rule: stacking more Eeveelutions than the opponent is good, but picking one while behind can empower the opponent's shared Eeveelution passive more than your own team. Ban scoring gets a small extra bump against enemy Eeveelution stacks, especially for carry/assassin/healer Eeveelutions.
- The game exposes `game_view::compute_champion_positions(...)`, which returns the client `champion_positions` map from champion statistics. Directly seeding `Database.champion_patch_statistics` from `on_server_start` or `before_management_tick` caused Loading Game / Start New Game CTDs. A controlled `after_management_tick` experiment in `src\pokemon_position_seed.rs` was gated behind `logs\enable-position-seed.txt`; it allowed Start New Game to load, but crashed when advancing a day. A later companion-exe trigger (`PokemonMobaPositionSeedPatcher.exe`) that wrote an app-data flag and asked the mod to seed on save load also caused crashes when loading saves and creating new games. All position-seeding code, flags, logs, and generated patcher files were removed on June 7, 2026. Do not recreate this path unless the SDK adds a safe public position-stat seed API, a native champion position field, or a confirmed safe save/database mutation lifecycle.
- SDK probe against `DataChampionInfo` found no `positions`, `position`, `main_position`, `role`, or `sub_category` field, and `ModChampionInfo` has no equivalent method in the 0.4.9 API.
- 2026-05-29 crash isolation: Windows crash records pointed at `pokemon_moba.dll` offset `0x39d24`, landing in Rust drop glue for boxed native actions during teardown. Disabling both native AI hooks made the game launch; re-enabling only `PokemonDraftScoreHook` stayed stable. `PokemonMobaInputAi` was then narrowed with `matches()` so it attaches only to Eevee, Emboar, and Skeledirge, and it now returns replacement inputs only after `ctx.is_valid_input()`.
- The draft hook assumes vanilla champions occupy indices `0..60` and native Pokemon follow registration order from `register_champions()`. If another mod or a game update changes champion indexing, revisit `VANILLA_CHAMPION_COUNT` and the candidate-index mapping.
- The title-screen simulated battle may still sample from the full base champion database before save/server filtering. If that remains visible, treat it as a separate showcase-layer problem rather than proof that actual match availability is broken.
- Item modding is parked. The SDK can register custom passive items, but base items remain visible/purchasable and no exposed item scoring, purchase-filter, shop-filter, or item recommendation hook has been found. Direct `Database.item_setting` mutation through `set_server_extension(...)` caused a Start New Game hard crash, and overriding the base item icon atlas is unsafe while base items remain present. Do not re-add item tooling or live item registration unless a safe data-layer replacement or real shop/item hook is found.

## Day-Advance Stability Rules

On June 6, 2026, repeated advance-day hard crashes and soft hangs were isolated to base-engine `attempt to divide by zero` panics during management simulation. The mod callback panic probes stayed empty, so the problem was the engine consuming unsafe SDK-facing metadata rather than a Pokemon move callback panicking directly. After clamping action metadata and flooring all-zero AI expected damage, the user simulated at least 100 games across several day advances with no crash. A few Pokemon were not selected in that run, so keep this rule set in place rather than treating any one Pokemon as proven safe by omission.

- Keep `PokemonAction`'s engine-facing wrappers: `duration`, `cooltime`, `start_timing`, `range`, and `growth_range` must never return zero to the SDK. Source move constants may still use `growth_range: 0`, but `ModEffect.growth_range` must be clamped before the engine sees it.
- Do not register placeholder no-op actions. Avoid `PokemonMoveEffect::None` for real attack/skill/skill2/ult slots. If a Pokemon concept has no true second move, give it a long-cooldown reserve/status action with valid targeting and non-zero metadata, as was done for Delibird's temporary `Present (Reserve)`.
- AI expected-value methods must not report zero-valued `Some(...)` metadata. `expected_cc_time()` should filter out `Some(0)`, `expected_buff()` should filter out zero-duration or empty `BuffState`, and `expected_move_distance()` should return `None` for zero ticks, zero speed, or zero distance.
- `expected_damage()` must not return `(0, 0)` for a real registered action. The current compatibility floor reports 1 point of AI-only expected damage in a bucket matching the action category when the authored mechanic has utility/healing/movement but no direct damage. This does not change combat damage; it only prevents base management AI from dividing by an all-zero valuation.
- When adding unusual status, terrain, healing, shield, movement, channel, reserve, or self-targeted moves, verify their SDK metadata path as carefully as their combat behavior. The base engine may query metadata during management simulation for champions that are not selected into the visible match.
- Keep the static `/` and `%` audit discipline in `pokemon_content.rs`: variable denominators should use `.max(1)`, `saturating_div`, explicit non-zero checks, or non-empty collection checks before modulo.

## Damage Model

- Pokemon and moves both have Pokemon types.
- STAB applies when move type matches one of the Pokemon's own types.
- Type effectiveness applies to damage taken by the defender.
- Super effective: `1.2x`
- Not very effective: `0.8x`
- Full immunities are not used; every attack can still deal damage.
- Dual-type multiplication is allowed, so double weakness/resistance can happen.
- Physical moves usually map to AD damage/scaling.
- Special moves usually map to AP damage/scaling.
- Status moves usually deal no direct damage, but can apply DOT, CC, healing, buffs, debuffs, terrain, shields, or other utility.
- Crit rule: only basic attacks can crit by default. Spell skills cannot crit unless the skill explicitly says it can. Tick damage, DOT, HoT, terrain pulses, and similar periodic effects cannot crit.

## Native Stat Reporting Bridge

As of June 10, 2026, Pokemon-layer combat stats have a working live bridge into the base game's normal stat persistence path. Do not discard this path or return to direct database/replay patching.

- Safe injection point: mutate the live combat player's `GamePlayer.statistics` and `GamePlayer.last_statistics` through `PlayerState.info` while still inside combat simulation. The current implementation lives in `mod\pokemon_moba\src\pokemon_status.rs`, especially `sync_pokemon_player_stats_to_base_player(...)` and `apply_pokemon_stat_deltas_to_base_player(...)`.
- Access path: `ctx.get_player(player_id)` gives a `PlayerRef`; `player.handle().as_ptr()` points at the internal `game_core::PlayerState`. Cast that pointer only transiently, and only after checking it is non-null.
- Required identity guard before writing: after casting to `PlayerState`, verify `player_state.info.id == player_id`, `player_state.info.team == player.team()`, and `player_state.info.position == player.position()`. If any check fails, skip the sync. Do not hold this pointer across callbacks or store it.
- Fields confirmed by `scratch\game_core_docs`: `PlayerState.info` is `GamePlayer`; `GamePlayer` owns `statistics` and `last_statistics`; `PlayerStatistics` has `deal`, `tank`, `heal`, `kill`, `death`, and `assist` fields. The persisted aggregate stat structs use different names (`dealing`, `tanking`, `healing`, `kills`, `deaths`, `assists`), so do not mix the two layers.
- Current working write policy: add custom ledger deltas to `statistics.deal`, `statistics.tank`, `statistics.heal`, `statistics.kill`, and `statistics.assist`, and apply the same deltas to `last_statistics`. This lets the native engine keep any native damage/kills/healing it already recorded while Pokemon custom events fill the missing totals.
- Deaths are intentionally not written by the Pokemon bridge. In-game testing showed native death tracking is already reliable, while custom death sync caused overcounts such as a single death becoming four deaths. Keep `add_pokemon_death_for_player(...)` updating the custom ledger for diagnostics, but do not call the base-stat sync from death updates and do not mutate `statistics.death` or `last_statistics.death`.
- Double-application prevention: `POKEMON_BASE_STAT_SYNC` records the last custom totals applied per `(combat_ctx_id, player_id)`. The bridge computes deltas from current `POKEMON_COMBAT_STATS` totals and only applies new increments. This is necessary because combat hooks may call sync repeatedly for the same player during one match.
- Context key: `combat_ctx_id(ctx)` uses `ctx.seed()` when available, otherwise the `GameCtx` pointer. Keep base sync and custom combat rows keyed by combat context plus player id so repeated day-advance simulations and reused entity/player IDs do not bleed into each other.
- Custom ledger source: `POKEMON_COMBAT_STATS` stores per-combat totals for damage dealt, damage taken, healing, kills, deaths, and assists. `record_pokemon_tracked_damage(...)` updates damage after Pokemon-layer damage is applied; `record_pokemon_kill_credit(...)` updates kills/deaths/assists when lethal Pokemon-layer damage is observed; healing paths call `add_pokemon_healing_for_player(...)`.
- Persistence mechanism: the mod does not write replay rows, athlete rows, champion aggregate rows, competition rows, or save data directly. It mutates the live `GamePlayer` stat fields during combat; the base game then appears to persist those values through its normal match/replay/stat pipeline. This is the safe path that made damage, damage taken, healing, and kills appear correctly in the normal statistics screens.
- Unsafe/crash-prone paths already tested: direct `MatchReplayData` mutation, athlete aggregate mutation, competition aggregate mutation, `champion_patch_statistics` writes, `ClientDatabase` stat patching, and broad `ModServerExtension` stat writes during save load/day advance caused crashes, bad data, or context bleed. Do not revive those paths unless the SDK exposes a documented safe lifecycle/API.
- Assists are currently wired to the same live stat sink if the custom assist ledger increments, but in-game testing still shows assists as zero. Treat assist work as a separate investigation into whether `award_pokemon_assists(...)` is populating `POKEMON_COMBAT_STATS.assists`, whether the base game persists `assist` from the live fields, or whether a native assist-specific event path is required.
- Hard protection rule: do not edit the Pokemon damage resolver, custom combat ledger, or native stat bridge without explicit request and permission from James. The current behavior is hard-won and should be treated as a stable integration point, not an area for opportunistic cleanup or refactor.

Offline save editing is not currently viable with SDK 0.4.9. TFM2 saves are `TFM2` plus gzip payload, but `Database::import_from_path` / `Database::load` fail on modern modded saves before reaching target fields, e.g. `original_info_sheet ... PlagueDoctorChampionInfo ... tags ... unknown enum kind: 100/900`. This means a standalone tool cannot safely deserialize, mutate, and reserialize the whole save with the public SDK. Raw binary patching of compressed save internals is not acceptable for normal mod support.

## Baseline Stat Rules To Preserve

- Our native Pokemon damage should continue to flow through `ctx.deal_damage` so engine-level defense, magic resistance, shields, incoming damage modifiers, and penetration have the best chance of applying normally.
- Direct Pokemon basic attacks should read `EntityStat.crit_chance` and apply the Pokemon crit rule in our layer. Shell Armor blocks these Pokemon-layer crits.
- Attack, AP, HP, defense, MR, move speed, HP regen, and crit chance are visible through `EntityStat` and can be used directly by Pokemon move scaling.
- Native `ModChampionInfo` exposes `stat()` and `growth()`, so base Pokemon stats are visible to the game and to normal runtime stat modifiers.
- Native `ModAction` exposes `cooltime(&EntityStat, level)`, so action cooldowns should keep reading the caster stat where possible. Engine cooldown modifiers are expected to apply around that value, while Pokemon-layer custom cooldowns such as Power-up Punch must remain explicitly tracked in `pokemon_status.rs`.
- The vanilla patcher-facing `PatchableFields` methods exist on the internal/base `ChampionInfo` trait, but they are not exposed by the native `ModChampionInfo` trait. That means custom native move constants such as burn chance, special status duration, execute width, charge scaling, and passive percentages are not automatically mutable by the built-in champion patcher unless a deeper/undocumented hook is found later.
- For compatibility, keep Pokemon move numbers centralized in Rust effect structs and formatter helpers. Runtime behavior and tooltip text should read the same values. Later, a Pokemon balance registry can feed these constants from a mod-owned config/save layer or generated data file, but it will be our compatibility layer rather than the vanilla patcher's `PatchableFields` system.
- Attack speed, cooldown reduction, vamp, heal reduction, range, radius, max-HP bonus damage, DOT amplification, and base/skill damage reductions live on `BuffState`. Effects can inspect active buffs through `EntityRef::buff_at`, but `ModAction::cooltime()` and `ModAction::effect()` only receive `EntityStat`, so dynamic cooldown/range support needs in-game verification or explicit Pokemon-layer handling.
- Base-game item stats can still affect Pokemon runtime stats. Pokemon-layer mechanics should continue to treat observed item-like stat surplus conservatively; a full custom item system is parked until base items can be hidden or suppressed safely.
- SDK examples use positive `skill_cooldown_mult` / `ult_cooldown_mult` for cooldown reduction. Keep Pokemon cooldown-reduction buffs positive unless in-game testing proves the sign is inverted.
- Pokemon-authored timed stat buffs and debuffs should go through `pokemon_status::add_beneficial_buff` or `pokemon_status::add_harmful_buff`, not raw `ctx.add_buff`, when they should be visible to Pokemon-layer mechanics. The wrapper records source, target, duration, polarity, and transferable `BuffState` fields in a ledger. Starmie's Heart Swap uses this ledger to move cleanly invertible stat buffs/debuffs by applying a counter-buff to the original holder and reapplying the original effect to the new target. Heart Swap also transfers custom Pokemon-layer harmful states from Starmie to the target, including Burn, Poison, Bleed, Infestation, Paralysis, Frozen, Confusion, Soak, Spirit Shackle, Leech Seed, Anti-Heal, Octillery Lock-On, pending Nasty Plot, and Arbok Miasma stacks. Miasma is not a DOT; it is transferred as stack state while preserving source, stack count, poison-award count, and remaining duration. Heart Swap also steals supported custom protective states from the target, including Endure, Audino Protect, Reversal, soft untargetable, Counter, Detect, Charm healing, and Gallade Prediction. Native engine-only flags that cannot be cleanly inverted, such as existing damage reduction/immunity, native `heal_reduce`, and native `cc_immune`, may still linger on the original holder until their engine timer expires unless represented by one of our custom states.
- Custom item replacement is not active. A compile probe showed `ItemSetting` fields are syntactically mutable, but live mutation caused a new-game CTD. Current custom item assets, templates, and scripts have been removed; revisit only after finding a safe item database/shopping hook.
- Custom channel/cooldown mechanics are viable through Pokemon-layer state plus `ModPlayerInputAi`. Use a short engine cooldown only as a shell, track real ready times in `pokemon_status.rs`, and have the AI wrapper suppress casts or intentionally move to release/cancel channels.
- Ultimate cooldown scale: author ult cooldowns in real seconds, not skill-like cooldowns. Extremely short ults should be about 30s, ordinary ults should usually land around 50-75s, high-impact long ults around 90s, and map/global/execute/teamfight-warping ults can reach 110-120s. At 60 ticks per second this means ordinary ult cooldowns are usually 3000-4500 ticks. For custom shell actions such as Emboar Power-up Punch or Dedenne Electric Terrain, keep the technical engine cooldown short only when needed for release/recast behavior, but expose and enforce the real cooldown through the Pokemon-layer custom cooldown and tooltip summary.

## Champion Categories

Use game categories, not user-facing shorthand:

- User says Caster/Mage -> use `ChampionCategory::Magician`
- User says ADC/Ranged -> use `ChampionCategory::Range`
- User says Bruiser/Melee -> use `ChampionCategory::Melee`
- User says Tank -> usually use `ChampionCategory::Melee` plus `ChampionTag::Tank`; the SDK does not have a Tank category.
- User says Support/Utility -> use `ChampionCategory::Util`
- User says Assassin/Jungler -> use `ChampionCategory::Assassin`

Tags should reflect gameplay: `AP`, `AD`, `Magic`, `Range`, `Melee`, `Tank`, `CC`, `Dot`, `Heal`, `Shield`, etc.

## Existing Custom Statuses

- Paralysis: custom tracked status; periodically rolls to stun. Electric/Ground/Rock immune.
- Burn: stackable magic DOT. Fire/Ground/Water immune.
- Poison: stackable magic DOT. Poison/Grass/Steel immune.
- Bleed: physical DOT.
- Frozen: engine stun + skill block. Ice immune.
- Burn, Poison, Frozen, Paralysis, Bleed, and Arbok Miasma should apply to any alive non-tower entity, including lane minions and neutral camps. Do not gate these statuses behind champion/Pokemon-only checks unless a specific move explicitly requires Pokemon-only behavior.
- Leech Seed: tethered DOT/heal, can spread through the tether line.
- Confusion: custom stack system for Pokemon actions only; redirects or reverses the next Pokemon action.
- Soft untargetable: direct Pokemon targeted actions skip the entity, but line/AoE/collision can still hit.
- Misty Terrain: persistent field that heals allies and slows enemies.
- Charm Heal: heal-over-time on an allied champion.
- Spiked Hide: Feraligatr reflect. Pokemon champion reflect is gated by a hit ledger recorded from our physical base-attack damage path; lane minions and jungle camps still trigger reflect automatically.
- Shell Armor: Torterra blocks Pokemon-layer critical damage such as Mega Launcher crits and Confuse Ray forced crits. The current SDK does not expose an incoming base-engine crit flag, so this may not block hidden vanilla crit resolution if it happens outside our damage layer.
- Eevee Wish: custom channel. It resolves on movement or max channel duration. The Pokemon AI wrapper now deliberately breaks the channel at partial charge thresholds so the heal can release early instead of always waiting for max duration.
- Emboar Power-up Punch: custom channel and custom cooldown. Starting the ult opens a Pokemon-layer charge; movement or max channel releases it. A hit resets the custom cooldown, while a miss applies the full custom cooldown. The engine cooldown is intentionally short and only prevents immediate technical recast spam.
- Greninja Quick Attack: custom scheduled return dash. The outward dash/hit uses the native force-move path and a delayed Pokemon-layer state applies a return force-move toward the cast origin.
- Greninja Battle Bond: custom kill-stack state. Enemy Pokemon kills increase Water Shuriken damage in Pokemon damage logic and add positive `ult_cooldown_mult` buffs to reduce Water Shuriken cooldown. Verify in-game that the SDK `on_kill` target argument only fires for champion kills as expected.
- Decidueye Long Reach: successful Leafage hits add short timed `range` buffs, approximating consecutive-hit range growth. If Leafage stops landing, the timed buffs fall off; true instant reset-on-miss would need a direct buff removal hook.
- Decidueye Spirit Shackle: custom status prevents AI-controlled Pokemon from choosing `Input::Return` for 2 minutes or until death. This blocks recall/return decisions through our Pokemon AI wrapper; verify later whether manual player return input can also be intercepted at a lower layer if needed.
- Soak: custom timed status currently introduced by Inteleon. Water-type Pokemon damage against soaked targets treats the defender as Water-weak, preserving Water STAB while overriding the defender's usual type resistance/weakness for that Water hit.
- Skeledirge Will-O-Wisp charges: native action uses a short technical cooldown, while Pokemon-layer state tracks 3 charges and recharges one charge every 420 ticks. The AI wrapper suppresses Skill2 when no charges remain.
- Porygon-Z type state: Pokemon-layer state tracks current type and unique type mask. Conversion copies champion target primary type; Conversion 2 changes to a random currently super-effective type; Recover resets to Normal. Porygon-Z damage uses the current type for Conversion moves, while Tri Attack uses a fixed forced STAB + super-effective multiplier and can crit.
- Blissey Healer: registered Blissey heals roll 50% on allied non-self heals. On success, harmful Pokemon-layer statuses are cleansed; if none are present, the target receives a short movement speed buff. Heal Pulse is a moving caster-centered aura, Softboiled reuses the HoT path on self, and Healing Wish heals all allied Pokemon globally for 75% max HP.
- Kleavor Sharpness/Stone Axe: consecutive damaging hits are tracked in Pokemon-layer state; Sharpness grants +5% Pokemon-layer crit chance per consecutive hit after the first, capped at +50% with a 5-second window. Stone Axe separately applies Bleed every third hit. Stealth Rock uses a defense buff plus alternating soft-untargetable windows so direct Pokemon targeting skips Kleavor while AoE/DOT can still hit.
- Scizor Light Metal: Pokemon-layer passive tracks distance moved since last taking or dealing damage. After 120,000 world units, Scizor gains 6 seconds of temporary HP equal to 20% of current movement speed. The SDK does not expose a separate shield resource, so this is implemented as a temporary HP buff and can look like a health increase in game. Swords Dance uses the current movement speed stat directly to grant critical chance, and Pokemon-layer critical chance above 100% is converted into critical damage.
- Ursaluna Blood Moon: Pokemon-layer state applies attack, attack speed, vamp, armor, and magic-resistance buffs, drains HP per second, blocks Pokemon-layer crits, and prevents enemy type resistance from reducing Ursaluna's Pokemon damage below 1x. The normal Ursaluna sheet keeps regular animations for idle/run/basic skills, but its `ult` animation tag uses the Bloodmoon Ursaluna sprite frames.
- Form-switch experiment: a temporary data-driven `Stance Probe` champion confirmed `DataEffectDef::SwitchByBuff` plus `view_buffs` can visibly maintain an alternate Demon/Archfiend form while attacking. This makes data-driven stance/form visuals viable for future stance Pokemon; native Pokemon can mirror the behavior by adding/removing a form buff and branching skill effects while the buff is present.

## Repeatable Pokemon Intake

User can provide casual templates. Convert them into native champion entries and new effect variants only when existing ones cannot express the mechanic.

Required fields:

- Pokemon name
- Types
- Role/category
- Range feel
- Damage profile
- Stats vibe
- Basic attack: name, type, category, targeting, effect, cooldown, range, scaling
- Skill 1: same fields
- Skill 2: same fields
- Ultimate: same fields
- Passive: name and behavior
- Notes
- Draft strategy traits, threats, and vulnerabilities

Draft strategy damage tags:

- Use `AdDamage` for Pokemon that are meaningful physical/attack-damage contributors.
- Use `ApDamage` for Pokemon that are meaningful special/ability-power contributors.
- Mixed damage Pokemon can use both when both profiles matter to draft composition.
- Pure healers and utility tanks with only token damage can omit both so the draft scorer does not treat them as real team damage sources.

Implementation checklist:

1. Receive or create custom sprite assets under `assets\custom_spritework\champions\<short-id>#sheet.png` and `assets\custom_spritework\champions\<short-id>#anim.fanim`.
2. Render transparent preview GIFs with `tools\render-sprite-previews.py` and QC framing/action readability before syncing.
3. Run `tools\sync-custom-champion-sprites.py` to copy staged files into `mod\pokemon_moba\champions_custom` and normalize `mod.override_info` remaps.
4. Add or reuse `PokemonMoveEffect` variants in `pokemon_content.rs`.
5. Add persistent state helpers in `pokemon_status.rs` if the move needs DOT/HoT/terrain/custom tracking.
6. Add the `PokemonChampion` constant and include it in `POKEMON_CHAMPIONS`.
7. Add the localized champion name in `mod/pokemon_moba/text/champion.i18n` under `description.pokemon_moba_<short-id>.name`. Missing this causes the UI to show raw keys such as `description.pokemon_moba_<id>.name`.
8. Add lane intent to `POKEMON_POSITIONS`.
9. Add draft traits, threats, and vulnerabilities to `POKEMON_STRATEGIES`.
10. Add the display name to `POKEMON_NAMES` in `pokemon_ui.rs`, keeping the same order as `POKEMON_CHAMPIONS` and `POKEMON_POSITIONS`.
11. Add custom AI wrapper logic in `PokemonMobaInputAi` if a move needs early channel release, custom cooldown suppression, or a non-standard decision layer.
12. Add action descriptions.
13. Add passive hooks where needed, and make an explicit Passimian Receiver compatibility decision for the passive. If the passive can sensibly apply to Passimian, wire it into Receiver predicates/update/damage/kill hooks; if not, document why it is structurally incompatible. Attach/tether passives such as Clawitzer and Comfey are structurally incompatible with Receiver; Wonder Guard is structurally incompatible because it depends on Shedinja's 1 HP shell state.
14. Route any new Pokemon-authored timed stat buff/debuff through the Pokemon buff ledger wrapper (`add_beneficial_buff` / `add_harmful_buff`) so Heart Swap, Technician-style ally-buff hooks, and future buff-aware mechanics can see it. Use raw `ctx.add_buff` only for untracked engine compatibility pulses, permanent stat grants, or effects that intentionally should not be copied/stolen.
15. Check the day-advance stability rules for every new action: no placeholder no-op slots, no zero engine-facing duration/cooldown/start timing/range/growth range, no zero-valued `Some(...)` expected metadata, no `(0, 0)` expected damage for real registered actions, and no unchecked variable division/modulo.
16. Run `rustfmt`.
17. Run `tools\build-native.ps1`.
18. Run `tools\deploy.ps1`.
19. Tell the user what is implemented, what is approximated, and what needs in-game testing.

- Active move descriptions and passive descriptions are generated from Rust values, not handwritten freeform copy. Passives are appended to the basic attack tooltip because the native `ModPassive` hook has combat events but no separate tooltip/description method.
