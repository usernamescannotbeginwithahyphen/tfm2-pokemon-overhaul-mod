# Pokemon Sprite Asset Pipeline

The game documentation covers PNG sprites, Aseprite files, and manual PNG animation sheets. It does not document direct GIF loading, so Showdown animated GIFs are converted into the manual animation pair that TFM2 expects:

```text
mod/pokemon_moba/champions/pikachu#sheet.png
mod/pokemon_moba/champions/pikachu#anim.fanim
```

Champion data or native registrations should refer to the base path:

```text
asset/pokemon_moba/champions/pikachu
```

## Shortcut Animation Strategy

Each converted Pokemon starts from one normalized animation loop, then a generated overlay pass gives the champion action tags distinct readouts:

```text
idle
run
attack
skill
skill2
ult
dead
```

This keeps the roster scalable while avoiding the worst readability problem: Pokemon no longer look like they are standing still during every action. `attack`, `skill`, `skill2`, and `ult` get Pokemon-specific glows, pulses, beams, arcs, elemental bursts, and action-shape readouts generated by:

```powershell
.\.venv\Scripts\python.exe .\tools\generate-champion-action-overlays.py mod\pokemon_moba\champions\pikachu
```

These are still sprite-attached effects, not map-positioned projectile/impact art. Native Rust champions currently have no documented API to spawn named `ViewEffect` or `ViewProjectile` assets directly, and `debug_draw_*` calls have not shown up in normal release gameplay.

## Static UI Icons

The native champion SDK does not document a separate portrait/icon hook for draft, news, comms, kill feed, or balance panels. Those UI surfaces appear to reuse the champion animation asset, usually the ordinary `idle`/common tags. An experiment remapping the unsuffixed champion base path to static PNGs caused crashes because some engine paths still expected a full animation asset. Keep the combat `#sheet` and `#anim` remaps as the reliable path for native champions.

The generator also emits a centered transparent 96x96 static icon for every Pokemon:

```text
mod/pokemon_moba/icons/champions/pikachu.png
```

These static icon files are kept for later custom UI work, but they are not currently remapped onto native champion base assets.

## One Pokemon

```powershell
.\tools\convert-pokemon-sprite.ps1 -Name pikachu
```

The converter also runs the action overlay/icon pass, so every newly added Pokemon should end with:

- generated `idle`, `run`, `dead`, `attack`, `skill`, `skill2`, and `ult` tags
- `mod\pokemon_moba\icons\champions\<short-id>.png` as a centered static portrait asset

## Batch Cache And Convert

```powershell
.\.venv\Scripts\python.exe .\tools\showdown_sprites.py index
.\.venv\Scripts\python.exe .\tools\showdown_sprites.py download --names pikachu charizard blastoise venusaur
.\.venv\Scripts\python.exe .\tools\showdown_sprites.py batch-convert --names pikachu charizard blastoise venusaur --output-dir mod\pokemon_moba\champions
```

Use `--limit` with `download` or `batch-convert` when testing broad conversions.

## Default Sizing

The converter crops the union of all visible GIF frames, scales the Pokemon into a 96x96 transparent canvas, fits visible content within 58 pixels, and bottom-centers champions with 24 pixels of padding. That bottom band keeps the game's health bars/nameplates from covering the Pokemon body while preserving the same combat position.

Large, small, or visually noisy Pokemon use per-Pokemon content sizes before the action overlay pass. These are gameplay-readable size classes, not exact Pokedex scale, because exact scale would make some Pokemon unusably large or tiny in the TFM2 lane view. Current important overrides:

- Very small/small examples: Pikachu 36px, Eevee 38px, Porygon-Z 42px, Umbreon 42px.
- Regular examples: Greninja 50px, Inteleon 52px, Decidueye 54px, Scizor 54px.
- Big/large examples: Charizard 68px, Feraligatr 68px, Torterra 72px.

The profile table lives in `tools/convert-pokemon-sprite.ps1`; future Pokemon should be added there when they are first converted.
